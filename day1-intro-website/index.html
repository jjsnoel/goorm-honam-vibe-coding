수업 철학

AI는 대신 만들어주는 마법이 아니라, 함께 완성하는 도구입니다

바이브코딩은 “AI에게 다 맡기는 것”이 아닙니다. 좋은 질문, 작은 수정, 반복 테스트를 통해 결과물을 완성하는 방식입니다.

동준상 강사는 10년 이상 AI·클라우드·오픈소스 분야에서 강연과 실무를 해오며 기술의 화려함보다 실제 문제 해결의 가치를 강조해 왔습니다. 입문자는 모든 코드를 외우기보다 구조를 이해하고 수정하는 경험을 먼저 쌓습니다.

AI는 목적이 아니라, 여러분의 아이디어를 웹페이지로 바꾸는 도구입니다.

“오늘의 목표는 완벽한 개발자가 되는 것이 아니라, AI와 함께 작동하는 첫 웹페이지를 완성하는 것입니다.”
수업 흐름

이해하기 → 따라 만들기 → 내 것으로 바꾸기

복잡한 개발 과정을 세 단계로 나누어, 오늘 안에 하나의 웹페이지를 끝까지 완성합니다.

1
👀

이해하기

HTML 문서 구조와 웹페이지 구성 요소(제목, 섹션, 버튼)를 눈으로 확인합니다.

2
🛠️

따라 만들기

AI 프롬프트를 활용해 섹션별 코드를 생성하고, 하나의 index.html에 붙여봅니다.

3
✨

내 것으로 바꾸기

문구, 색상, 프로필, 버튼을 자기 스타일로 수정해 나만의 페이지로 확장합니다.

실습 도구

오늘 실습에서 사용할 도구

코딩을 처음 시작하는 학습자도 익숙해질 수 있도록, 각 도구의 역할을 명확히 구분해 사용합니다.

💬 ChatGPT

아이디어 정리, 문구 작성, 오류 설명 요청에 사용합니다.

⌨️ Cursor

HTML 파일 작성·수정·미리보기 흐름을 한곳에서 진행합니다.

🤖 Claude Code

프로젝트 구조 점검, 코드 개선, 리팩토링 실습에 활용합니다.

🌐 Vercel / GitHub Pages

완성한 웹페이지를 온라인으로 공유하고, 포트폴리오 링크로 활용합니다.

도구를 많이 쓰는 것보다, 하나의 결과물을 끝까지 완성하는 것이 더 중요합니다.

오늘의 결과물

오늘 완성할 수 있는 결과물

바이브코딩은 코드를 외우는 수업이 아니라, AI와 함께 내 아이디어를 실제 웹페이지로 바꾸는 실습입니다.

Result 01

자기소개 페이지

이름, 관심 분야, 목표, 연락처를 담은 개인 페이지를 완성합니다.

Result 02

프로젝트 소개 페이지

문제 정의, 해결 아이디어, 사용 기술, 기대 효과를 소개하는 페이지입니다.

Result 03

강사·서비스 소개 페이지

지금 보고 계신 이 페이지처럼, 전문성·수업 방식·문의 버튼을 담은 랜딩 페이지입니다.

오늘의 완성 기준

  • 하나의 index.html 파일로 브라우저에서 바로 실행된다
  • 모바일과 PC에서 모두 읽기 편한 반응형 레이아웃이다
  • 최소 5개 이상의 섹션을 포함한다
  • 버튼 클릭 시 페이지 내부 섹션으로 부드럽게 이동한다
  • 본인의 문구와 색상으로 수정되어 있다
자주 묻는 질문

처음이어도 괜찮습니다

수업 전에 많이 받는 질문을 정리했습니다. 궁금한 점은 수업 중에도 편하게 물어보세요.

네. 오늘은 코드를 외우는 것보다 구조를 보고 수정하는 경험에 집중합니다. AI 도구와 함께 한 섹션씩 완성해 나가면 됩니다.
네. HTML 안에 CSS와 JavaScript를 함께 넣으면 싱글 파일 웹페이지를 만들 수 있습니다. 오늘 실습은 바로 그 방식으로 진행합니다.
오류 메시지와 현재 코드를 다시 AI에게 보여주고, 수정 요청을 반복하는 방식으로 해결합니다. 이 과정 자체가 바이브코딩의 핵심 실습입니다.
네. 문구, 색상, 섹션을 바꾸면 개인 포트폴리오나 프로젝트 소개 페이지로 확장할 수 있습니다. Vercel이나 GitHub Pages로 배포해 링크를 공유할 수도 있습니다.
